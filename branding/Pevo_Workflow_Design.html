03 핵심 원칙
📋
1. 기본은 계획 모드
사소하지 않은 모든 작업은 계획 모드로 시작. 3단계 이상이면 체크리스트 작성. 뭔가 잘못되면 즉시 멈추고 다시 계획.
🤖
2. 서브에이전트 활용
메인 컨텍스트를 깔끔하게 유지. 조사·탐색·병렬 분석은 서브에이전트에 위임. 하나당 하나의 작업만 — 집중 실행.
🔄
3. 자기개선 루프
수정을 받을 때마다 lessons.md에 기록. 같은 실수를 반복하지 않는다. 세션 시작 시 교훈 목록을 먼저 검토.
✅
4. 완료 전 반드시 검증
작동한다는 걸 증명하기 전까지 완료로 표시하지 말 것. "시니어 엔지니어가 이걸 승인할까?" 자문.
💎
5. 우아함 추구 (균형)
"더 우아한 방법이 있지 않을까?" 자문. 단순하고 명백한 수정엔 생략 — 오버엔지니어링 금지.
🔧
6. 자율적 버그 수정
버그 리포트를 받으면 그냥 고칠 것. 하나하나 물어보지 말 것. 로그·에러·실패한 테스트를 직접 찾아서 해결.
실행 규칙
단순함 우선
모든 변경은 최대한 단순하게. 건드리는 코드는 최소화.
게으름 금지
근본 원인을 찾을 것. 임시방편 없음. 시니어 개발자 기준.
최소 영향
꼭 필요한 부분만 수정. 새로운 버그를 만들지 말 것.
04 축적된 교훈 (Lessons Learned)
실수에서 배운 규칙들. 세션 시작 시 반드시 검토합니다. 현재 13개 교훈 축적.
PptxGenJS 관련 (5건)
L-001
mkShadow는 반드시 팩토리 함수로
PptxGenJS가 shadow 객체를 mutate → 매번 새 객체 반환하는 팩토리 사용
L-002
IMG 헬퍼에서 prefix 중복 주의
genIcon()은 base64만 반환, IMG() 헬퍼가 prefix 추가
L-003
팀 프로필 이미지 경로 검증
Glob으로 실제 파일 목록을 먼저 확인한 후 경로 지정
L-004
슬라이드 요소 대비(Contrast) 검증 필수
밝은 배경에는 bodyText(2D3748) 또는 subText(64748B). midGray는 dark 전용.
L-005
color 이름과 실제 명도 혼동 주의
hex 값의 실제 명도를 확인. 이름에 속지 말 것.
인프라/환경 관련 (4건)
L-006
WebFetch는 egress proxy에 의해 차단 가능
WebSearch 우선 사용. WebFetch 실패 시 대안 전환.
L-007
LibreOffice PDF 변환 시 한국어 폰트 미지원
VM 환경 제한. QA 시 한글 렌더링 이슈 무시. Content QA로 별도 검증.
L-008
soffice PDF 출력 경로 주의
변환 후 ls로 실제 출력 위치 확인 후 후속 작업.
L-009
병렬 도구 호출 시 첫 번째 호출 실패 가능
WebSearch는 단독 호출하거나 실패 시 재시도.
작업 흐름 관련 (2건)
L-010
컨텍스트 소진 관리
서브에이전트 적극 활용. 중간 결과물은 파일로 저장.
L-011
QA는 생성 직후 반드시 수행
PPTX → PDF → 이미지 → Visual QA + Content QA 파이프라인 항상 실행.
콘텐츠 관련 (2건)
L-012
재무 데이터 정확성 — ARR ≠ 매출
ARR은 구독 기반 반복 매출만. 일회성 매출은 별도 표기.
L-013
안전 표현 — "100% 보장" 금지
AI 정확도는 "95%+ 목표" 등 목표치로. "보장" 표현 절대 금지.
