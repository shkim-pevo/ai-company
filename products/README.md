# 영양제 · 일반의약품 · 전문의약품 제품 DB

> 관리 담당: 추천엔진팀 — 제품 데이터 큐레이터
> 검증 담당: 추천엔진팀 — 약학 도메인 전문가 + 퀄리티팀

## 폴더 구조

```
products/
├── README.md                  ← 본 파일 (관리 가이드)
├── supplements.csv            ← 영양제(건강기능식품) 제품 DB
├── otc_drugs.csv              ← 일반의약품(OTC) 제품 DB
├── rx_drugs.csv               ← 전문의약품(Rx) 제품 DB
└── data_dictionary.md         ← 컬럼 정의 및 코드 체계
```

## 데이터 관리 원칙

1. **CSV 포맷 표준**: UTF-8 인코딩, 쉼표(,) 구분, 큰따옴표("") 이스케이프
2. **다중 값 구분자**: 세미콜론(;) — ingredients, interactions, dur_tags 등
3. **신규 제품 등록**: 제품 데이터 큐레이터가 성분 분석 → 카테고리 매핑 → CSV 등록
4. **정보 부재 시**: 공인 출처(식약처 의약품안전나라, 건강기능식품 정보포털) 기반 자체 조사 → DB 보완
5. **변경 이력**: 모든 수정은 `updated_at`, `updated_by` 컬럼에 기록
6. **교차 검증**: 약학 도메인 전문가가 성분 상호작용·DUR 관련 정보 검증
