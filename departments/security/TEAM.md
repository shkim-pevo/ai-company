# 보안팀 (Security Team)

## 👤 팀장 프로필
- **이름**: 보안팀장
- **역할**: 보안 전략 수립, 팀 관리, 보안 검증 및 승인
- **보고 대상**: 리더십팀

## 🎯 팀 미션
시스템과 데이터의 보안을 보장하고 의료 데이터 규제 준수

---

## 👥 팀 구성

| 역할 | 담당자 | 핵심 업무 |
|------|--------|---------|
| **팀장** | 보안팀장 | 보안 전략 수립, 팀원 관리, 최종 보안 검증, 승인 |
| **보안 분석가** | 보안 담당 | 취약점 스캐닝, 침투 테스트, 보안 감사 |
| **개인정보 보호 담당** | 개인정보 담당 | PIMS 인증 관리, 동의서 관리, 데이터 비식별화 |
| **컴플라이언스 담당** | 컴플라이언스 담당 | 의료 데이터 규제, 약사법 준수, 감사 대응 |

---

## 📋 역할별 상세 책임

### 팀장 (보안팀장)
**책임 및 권한:**
- 기술팀(개발, DevOps)으로부터 보안 검토 요청 수령
- 보안 검토 계획 및 전략 수립
- 팀원(담당자)에게 보안 검토 업무 배정
- 각 담당자의 보안 검토 진행 상황 모니터링
- 최종 보안 검증 및 승인 판단
- 보안 이슈의 우선순위 결정
- 보안 현황을 리더십팀에 보고
- 보안 사고 대응 총괄

### 보안 분석가 (보안 담당)
**책임:**
- 애플리케이션 보안 코드 리뷰
- OWASP Top 10 취약점 진단
- 정적 분석(SAST) 도구를 활용한 취약점 스캔
- 동적 분석(DAST) 수행
- 침투 테스트(Penetration Testing) 실행
- 네트워크 보안 검증
- API 보안 검증 (인증, 인가, 레이트 제한)
- 취약점 분류 및 우선순위 지정
- 보안 개선 권고

**산출물:**
- 보안 코드 리뷰 결과
- 취약점 진단 보고서
- 침투 테스트 결과 보고서
- 보안 개선 권고사항

### 개인정보 보호 담당 (개인정보 담당)
**책임:**
- PIMS(Personal Information Management System) 인증 관리
- 개인정보 영향평가(Privacy Impact Assessment, PIA) 수행
- 환자 동의서 관리 및 법적 검토
- 의료 데이터 분류 및 태깅
- 데이터 비식별화 검증
- 데이터 암호화 검증 (저장, 전송, 접근)
- 개인정보 접근 통제(Access Control) 검증
- 개인정보 삭제 및 보존 정책 수립
- GDPR, 개인정보보호법 등 규제 준수

**산출물:**
- PIMS 인증 문서
- 개인정보 영향평가 결과
- 동의서 템플릿 및 관리 기록
- 개인정보 보호 정책 문서
- 데이터 비식별화 검증 결과

### 컴플라이언스 담당 (컴플라이언스 담당)
**책임:**
- 의료 데이터 규제 준수 (의료법, 의료기기법 등)
- 의약품 관련 규정 준수 (약사법 등)
- 보안 정책 수립 및 유지보수
- 보안 인증 관리 (ISO 27001, 의료기기 인증 등)
- 보안 감사(Security Audit) 계획 및 수행
- 보안 교육 및 훈련 프로그램 운영
- 보안 사고 대응 계획(Incident Response Plan) 수립
- 감사(Audit) 및 컴플라이언스 보고

**산출물:**
- 보안 정책 문서
- 인증 관리 기록
- 감사 계획 및 보고서
- 컴플라이언스 체크리스트

## 🔄 업무 프로세스

### 단계 1: 팀장 배정 (Assignment)
```
FROM: 개발팀장/DevOps팀장
TO: 보안팀장

애플리케이션: [앱명]
검토 대상:
- 소스 코드
- API 설계
- 인프라 설정
- 개인정보 처리

검토 항목:
- 인증/인가 (Authentication/Authorization)
- 데이터 암호화 (Encryption at rest/in transit)
- API 보안 (Rate limiting, Input validation)
- 네트워크 보안 (SSL/TLS, Firewall)
- 개인정보 보호 (PII, PIMS)
- 규제 준수 (의료법, 약사법 등)

타겟 보안 표준:
- OWASP Top 10 준수
- CWE (Common Weakness Enumeration) 기반 검사
- ISO 27001 기준
- 의료기기 보안 기준

관련 컴포넌트:
- 약사 대시보드
- 환자 앱
- 약물 상호작용 AI 모델
- POS 시스템 연동 API
```

**팀장 업무:**
- 보안 검토 요청 분석
- 위험도 평가 (Risk Assessment)
- 보안 검토 범위 결정
- 팀원(담당자)별 업무 배정
- 검토 일정 산정
- 필요시 외부 보안 전문가 참여 결정

### 단계 3: 팀장 검토/검증 (Review & Verification)

**보안 검토 결과 검증:**

1. **취약점 분류 및 우선순위 검증**
   - Critical: 서비스 가동 불가, 즉시 수정 필수
   - High: 데이터 유출 위험, 출시 전 필수 수정
   - Medium: 보안 약화, 우선순위 판단
   - Low: 개선 권고

2. **규제 준수 검증**
   - OWASP Top 10 준수 확인
   - 의료법 규정 준수 확인
   - 약사법 규정 준수 확인
   - 개인정보보호법 준수 확인
   - PIMS 인증 요구사항 충족 확인

3. **개인정보 보호 검증**
   - 환자 데이터 암호화 ✅
   - 접근 통제 구현 ✅
   - 감사 로깅 ✅
   - 동의서 완비 ✅
   - 데이터 비식별화 ✅

4. **기술 보안 검증**
   - API 보안 (인증, 인가, Rate limiting) ✅
   - 데이터 암호화 (저장, 전송) ✅
   - 네트워크 보안 (SSL/TLS, Firewall) ✅
   - 의존성 보안 (취약한 라이브러리 없음) ✅

**검증 결과:**
