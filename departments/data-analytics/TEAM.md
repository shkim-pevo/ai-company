# 데이터 분석팀 (Data Analytics Team)

## 👤 팀장 프로필
- **이름**: 데이터 분석팀장
- **역할**: 데이터 기반 인사이트 도출 및 의사결정 지원
- **보고 대상**: 리더십팀

## 🎯 팀 미션
데이터를 통한 비즈니스 의사결정 지원 및 성과 최적화

## 👥 팀 구성

| 역할 | 담당자 | 책임 영역 | 상태 |
|------|--------|---------|------|
| 팀장 | 데이터 분석팀장 | 전체 분석 전략, 대시보드 | 활성 |
| 비즈니스 분석가 | Business Analyst | KPI 대시보드, 매출/전환율 분석, 경영진 리포트 | 활성 |
| 프로덕트 분석가 | Product Analyst | 사용자 행동, 퍼널 분석, A/B 테스트 해석 | 활성 |
| BI 엔지니어 | BI Engineer | 데이터 파이프라인, ETL, 시각화 도구 | 활성 |

## 📋 역할별 상세

### 팀장의 책임
- 데이터 수집 및 파이프라인 구축
- 데이터 분석 및 시각화 전략
- 대시보드 구축 및 관리
- 성과 지표 모니터링
- 예측 모델링
- 인사이트 리포트 작성
- 조직 전체 데이터 기반 문화 구축

### 비즈니스 분석가
**책임:**
- KPI 대시보드 설계 및 관리
- 매출, 수익성 분석
- 전환율(Conversion Rate) 분석
- 비용 효율성(ROAS, CAC 등) 분석
- 경영진 리포트 작성
- 비즈니스 성과 지표 모니터링

### 프로덕트 분석가
**책임:**
- 사용자 행동 분석 (User Journey, Heatmap)
- 퍼널 분석 (Funnel Analysis)
- A/B 테스트 설계 및 결과 해석
- 사용자 세분화 (Segmentation)
- 프로덕트 성과 메트릭 분석
- 기능 개선 권고안 제시

### BI 엔지니어
**책임:**
- 데이터 파이프라인 구축 및 관리
- ETL(Extract, Transform, Load) 프로세스 개발
- 데이터 웨어하우스 설계 및 유지보수
- 시각화 도구(Tableau, Looker 등) 구축
- 데이터 품질 관리
- 대시보드 개발 및 자동화

## 🔄 업무 프로세스

```
팀장 배정
    ↓
담당자 실무 (데이터 수집, 분석, 시각화)
    ↓
팀장 검토/검증
    ↓
보고 (리더십팀, 요청팀)
```

### 1. 분석 요청
```
FROM: 리더십팀/기획팀/제품팀
TO: 데이터 분석팀장

분석 주제: [...]
질문: [답변이 필요한 구체적 질문]
데이터 소스: [...]
우선순위: [High/Medium/Low]
기한: [...]
```

**담당자 실무:**
- 비즈니스 분석가: 사업 영역 분석
- 프로덕트 분석가: 사용자/기능 분석
- BI 엔지니어: 데이터 추출 및 시각화

### 2. 데이터 분석
- 데이터 수집 및 정제
- 탐색적 분석 (EDA, Exploratory Data Analysis)
- 통계 분석
- 시각화
- 인사이트 도출

**팀장 검증:**
- 분석 방법론 검증
- 결과 타당성 확인
- 권고안 검토

### 3. 보고 및 인사이트 제시
```
FROM: 데이터 분석팀장
TO: 리더십팀/요청팀

## 분석 리포트

분석 목적: [...]

주요 발견사항:
1. [인사이트 1]
   - 지표: [...]
   - 변화: [...]
   - 차트 [...]

2. [인사이트 2]
   - 지표: [...]
   - 변화: [...]
   - 차트 [...]

권장사항: [...]
```

### 4. 정기 KPI 모니터링 및 리포트
- 주간 KPI 리포트
- 월간 성과 분석
- 분기별 전략 리뷰
